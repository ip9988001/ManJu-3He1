## 一、 具体功能与战略作用（它是什么，干嘛用的）

### 1. 具体功能：【视听解耦】与【纯声音信号榨取】

原著单集文本（`第1章01分集.txt`）里，台词、内心戏、环境侧写和角色的物理动作（如：一屁股瘫坐在地）是混在一块的肉。
大模型在这一步的唯一功能就是担任“音频导演”**，执行**声肉脱水——把所有不需要配音员嘴巴读出来的视觉废话（动作、表情、机位暗示）全部扒掉，只榨取出 100% 纯粹的、耳朵能听到的声音信号，并将其分拣给对应的角色实名。

### 2. 战略作用：为自动化并发配音（TTS）提供“绝对无歧义”的执行枪子

* **格式化听书感**：彻底干掉原著里“某某说道”、“某某怒斥”等文本残留，将其转化为纯粹的戏剧对白。
* **驱动多线程配音脚本**：产出的规范文本能让本地 Python 脚本无脑遍历，闭着眼睛根据`角色名字`去本地字典里匹配对应的`声音模型 ID（Voice ID）`，根据`情绪参数`去调整配音引擎的速度和基频，实现音频碎块的自动渲染。

## 二、 如何操作？需要输入什么？（数据进门与物理调度）

### 1. 物理操作流线

本地 Python 脚本通过文件指针进入对应的集级子目录（如 `仙逆-第1章节-分集01/`），抓取本集原著，并在内存中把当前所需的参考背景打包，全量砸给大模型 API。大模型结算后吐出中文标签文本，Python 脚本在本地无脑将其转化为 `.md` 存盘。

### 2. 必须输入的 4 大核心素材

* **原料正文：【第 X 章 01 分集.txt】**
* *状态*：上一步（步骤 02）刚切下来的、属于这一集的纯原著小说正文（几百字到千字）。


* **后顾因果锁：【步骤 02 产生的章级主派工单中关于本集的控制常数】**
* *状态*：Python 脚本在发包前，从主派工单里抠出来的两个核心字段：`音频台词钩子文案`（规定了本集配音最后一句必须是什么）和 `当前临界资产`（规定了本集有哪些标准实名角色会出声）。


* **中观导演手册：【本轮十章级中观视听控制总纲.md】**
* *状态*：让大模型明确这一集在整场大戏里的情绪分贝。如果是大战章，旁白语速就要加快、声调要高亢；如果是过渡章，语速就要平缓。


* **系统最高法律：【步骤 03 专属全中文标签合同（System Prompt）】**
* *状态*：常驻底层的静态规则，焊死换算公式（1秒=4个字），并锁死输出的中文尖括号标签格式。

## 三、 按照什么要求？产出什么内容？（数据出门与红线把关）

### 1. 大模型必须严格执行的 4 大刚性要求（防翻车红线）

为了防止大模型自己脑补剧情或者吐出无法解析的噪点，系统设立 4 条铁血军纪：

* **纯声音红线**：`有效声音内容` 里，**严禁夹带任何动作动作描述**。例如：不允许出现 `王林：（惊恐地退后两步）谁？！`。必须洗成：`发音角色：王林；情绪音调：【极度惊恐】；有效声音内容：谁？！`。动作归下游的视觉车间管，声音车间不准越权！
* **角色实名归宗红线**：出声的角色名字必须严格使用资产表里的标准实名（如 `王林_少年初始态`、`墨大夫`）或通用的 `旁白`。
* **【零对白特判补丁】**：如果整集原著全是武打和环境描写，一句角色台词都没有。大模型强制将发音角色全量判定为 `旁白`，将战况精简为旁白声音内容，严禁不吐标签。
* **【多角色异口同声特判补丁】**：如果原著出现“众人齐声惊呼”。大模型严禁合并吐出 `发音角色：众人`（因为本地找不到众人的Voice ID）。必须将其强制归类为系统通用音色 `旁白_龙套群音`，或者拆成多行输出。
* **尾尾咬合红线**：本集最后一行配音的文本内容，必须与上一步定好的 `音频台词钩子文案` **100% 逐字对齐**，少一个标点符号直接视为不合规，强制熔断。

## 四、 产出物在下游有什么功能？什么作用？（如何落地变现）

本地 Python 脚本接收到上述纯文本后，用普通的 `.split()` 提取干货，在对应的声音车间目录下自动生成并写死最终的物理文件：

* **物理路径**：`仙逆-第1章节-分集01/01集单集声肉脱水与声卡图层提取/01集_全角色配音与情绪调教派工单.md`

该文件落盘后，直接触发下游的**两大自动化变现功能**：

### 1. 物理变现功能 A：自动焊死全片总时长与时间轴基础（Time Stamper）

Python 脚本读取第一行的 `有效配音总字数：280`，自动运行底层的刚性数学公式：


$$\text{全集视频总长度} = \frac{280 \text{ 字}}{4 \text{ 字/秒}} = 70 \text{ 秒}$$


脚本会在内存里把这集漫剧的最终时间轴卡死在 **70 秒**。后续的每一行配音、每一张分镜图片，在用 FFmpeg 合片时，都必须在这个总长范围内进行对齐和时间分配，彻底防止音画不同步。

### 2. 物理变现功能 B：多线程驱动 TTS 语音引擎一键乱轰（Batch Audio Renderer）

自动化音频生成脚本开机，无脑遍历这个 `.md` 文件的行节点：

* 读到 `发音角色：王林_少年初始态` $\rightarrow$ 脚本自动去本地的配音人字典里捞出王林的专属克隆声音 ID（如 `Voice_ID_WangLin_001`）。
* 读到 `情绪参数` 和 `录音文案` $\rightarrow$ 把语速、声调等控制参数作为变量，连同纯文本台词一起打包，通过 API 砸给 TTS 语音合成引擎（如 CosyVoice）。
* **物理结果**：几秒钟内，该单集目录下的 `audio/` 文件夹里，就会整整齐齐地躺着机器自动轰出来的、带极高人类情绪起伏的配音切片文件：`line_01.mp3`、`line_02.mp3`、`line_03.mp3`……
