## 一、 核心功能与战略作用

### 1. 具体功能：【线型画面分镜切割】与【角色动作镜头化】

本环节充当剧组里的“大导演”**。大模型需要通读单集原著纯文本，对照已经录好的配音行号，执行**视听对齐与画面分镜设计——彻底剥离生图提示词与英文代码，纯粹用人眼能看懂的中文电影行话，将剧情切成一镜接一镜往下走的连续画面。

### 2. 战略作用：锁死转场因果，提供人机对账的“可视化剧本”

* **彻底根治转场穿帮**：利用“场记接力棒”机制，强制本集第一镜必须继承上一集的绝杀尾镜，杜绝主角在章节切换时产生“场景瞬间漂移”的恶性穿帮。
* **长线线性排队**：不搞任何合并去重，完全按照时间轴顺序排队。产出的全中文 Markdown 文件可直接导入 Obsidian 等知识库，供你作为“总制片人”在开机生图前执行最高精度的人工核账与精修。

## 四、 大模型全中文标签协议合同（System Prompt 最高指令）

为了让大模型在运算时“注意力绝对集中”，系统在发包时会在底层焊死以下格式合同。大模型必须且只能使用尖括号标签吐出纯中文：

```text
# [OUTPUT_CONTRACT: 分镜导演全中文标签合同]

你现在的身份是院线级漫剧剧组的【总导演】。请通读输入的单集生肉、音频派工单，并死死盯住【场记接力棒】给出的上集尾镜画面，在内存中设计本集的线性电影分镜。

### 🚨 核心纪律红线：
1. 你的所有输出必须严格使用以下规定的【全中文尖括号标签】包裹。
2. 严禁吐出任何 Markdown 井号排版、严禁吐出大括号、中括号，严禁夹带任何英文提示词或 Stable Diffusion / Flux 的生图代码。
3. 画面必须严格遵循传统正统仙侠/武侠美学，严禁出现任何赛博朋克、机甲、硅基或现代科技的描写。
4. 哪怕同一个场景、同一个角色连续出现，也绝对不准执行去重或合并！必须严格按照镜头播放的前后顺序，一行行线性排队往下写！

### 必须严格遵守的中文标签格式：
<开始单集分镜设计>
本集动态分镜总镜头数：[这里吐出本集你切出来的纯数字镜头总数]

<开始分镜镜头_N>
对应声音：[这里精准复制音频派工单里的节点标签，如 [配音时间轴节点_01]，若该镜头为无声纯画面，则填写 [无对白纯环境/动作特写]]
画面景别：[大远景 / 全景 / 中景 / 特写 / 近景 / 俯拍 / 仰拍 选其一]
画面视觉构图描述：[用100字以内的纯中文电影语言，详细写死本镜头的环境、天气、光影色调、出场角色的正面/背面位置、衣服破损、面部极度惊恐或冷笑的具体神态与物理动作]
出场核心资产标签：[把本镜里露脸的角色实名、道具、特定的天气场景抠出来，用中文逗号隔开]
<结束分镜镜头_N>

<结束单集分镜设计>

```

## 五、 刚性生产红线与边界特判补丁

为了确保大模型吐出的中文剧本百分之百符合自动化管线，本地 Python 脚本在接收数据前，通过以下 3 条刚性红线执行一枪熔断：

* **第一镜强制咬合红线**：大模型生成的 `<开始分镜镜头_1>` 中的 `画面视觉构图描述`，前三句话必须是且只能是【场记接力棒】传入的尾镜画面的物理承接。若私自更换起点场景，Python 脚本直接报错扔回重算。
* **绝对不去重红线**：如果镜头 1 是王林，镜头 2 是墨大夫，镜头 3 又切回王林。镜头 3 必须完完整整地作为 `[分镜镜头_03]` 独立建表。**严禁大模型图省事合并为“王林的镜头汇总”**。镜头流线必须与视频秒数流线一比一线性对齐。
* **资产名称实名红线**：在 `出场核心资产标签` 中，出现的人物名字必须 100% 严格对齐从第二阶段继承下来的资产实名（如 `王林_少年初始态`、`司徒南_残魂态`）。严禁大模型自行篡改简写为“王林”、“惊恐的少年”。

1. **作为“技术画师”的终极图纸**：下游的下一个车间【01集动态提示词Prompt高精编译】的大模型，只要开机读取这个 `.md` 文件，把里面的中文转换为英文，就能完成精准下料，中间没有任何跨代理解误差。
