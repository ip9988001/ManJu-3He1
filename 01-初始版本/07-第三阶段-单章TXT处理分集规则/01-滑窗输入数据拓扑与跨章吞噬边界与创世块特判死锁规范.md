## 一、 本地 Python 上游预处理：基于上一章 JSON 的历史切肉算法

当流水线排产到第 $X$ 章（且 $X > 1$）时，本地 Python 脚本在将当期正文发给大模型之前，必须先执行上游"物理切肉"动作，防止上一章已经吞噬过的剧情被重复生产。

### 1. 物理切肉算法执行流程

1. **捞取指针**：Python 脚本自动读取本地硬盘中 `第X-1章_流水线生产总执行派工单.json` 的尾部节点。
2. **判定跨章状态**：检查 `跨章吞噬机制` 下的 `是否激活` 字段。
* 若为 `False`，则第 $X$ 章正文不做任何裁剪，全量保留。
* 若为 `True`，则提取 `吞噬终点句子` 字段中的全中文纯文本字符串（设为变量 `吞噬标记线`）。

3. **定位与截断**：Python 读取 `第X章.txt` 全量文本，利用字符串检索函数现场定位 `吞噬标记线` 的结束位置。
4. **落盘存盘**：代码直接将 `吞噬标记线` 之前的所有文字物理丢弃，将剩下的纯净干货保存为临时内存变量 `第X章_切剩的纯净正文`。

### 2. 物理切割代码逻辑映射表

为了防止字符串匹配因为一个换行符而找不到，系统在切肉时必须遵守以下刚性对齐规则：

| 校验流步骤 | Python 内部逻辑动作 | 异常边界兜底策略 |
| --- | --- | --- |
| **步骤 A：清洗去除噪点** | 将 `吞噬标记线` 字符串执行 `.strip()`，剔除前后所有的空格与系统换行符。 | 若提取出的标记线为空，直接熔断报错，停止流水线。 |
| **步骤 B：全文字符串检索** | 执行 `index = 原著全文.find(吞噬标记线)`。 | 若返回 -1（即原著里找不到这句话），说明原著文本被篡改，**立即触发系统报警**，转由人工介入。 |
| **步骤 C：计算切刀终点** | 计算切刀物理切断位置：`cut_point = index + len(吞噬标记线)`。 | 严格锁死包含该标记句的最后一个标点符号。 |
| **步骤 D：执行物理截断** | 执行物理截断：`切剩正文 = 原著全文[cut_point:]`。 | 切剩的正文作为当期核心原料，直接送入内存包。 |

## 二、 零号索引特判：`创世块（Genesis Block）` 虚构常数硬编码注入规范

当系统启动并处理全局第一章（`第1章.txt`）时，由于本地硬盘中绝对不存在 `第0章_派工单.json`，Python 脚本将无法建立"后顾接力棒"。

为了防止程序抛出 `FileNotFoundError` 崩溃，系统建立 **`创世块（Genesis Block）`** 强制注入机制。

### 1. 创世块注入拦截拓扑

```text
【Python 脚本开机】
       │
       ▼
 检查当前章号: index == 1 ?
       ├── [是] ──→ 绕过磁盘检索 ──→ 现场在内存中声明 [创世块中文常数群]
       └── [否] ──→ 正常检索磁盘 ──→ 载入 [第 X-1 章_派工单.json]

```

### 2. 创世块（Genesis Block）全中文硬编码常数定义

系统检测到第一章时，在内存中原地捏造并焊死以下全中文伪派工单数据结构，作为输入流的一部分直接砸给大模型：

> **【创世块全局常数配置（死锁标准）】**
> * **上一集分集ID**："创世集_零号索引"
> * **上一集音频台词钩子**："大幕拉开，万物初始。"
> * **上一集画面视觉钩子**："电影级全黑转场，冷调色，画面中无任何出场角色，无任何动作发生。"
> * **上一集残余环境白噪音**："-60dB（绝对静音状态）"
> * **跨章吞噬机制**：
> * `是否激活`：否
> * `吞噬终点句子`："前情空白"

大模型接收到这一组全中文常数后，将自动识别出当前为全书的起点，从而放弃任何历史焊缝对接逻辑，直接从第 1 章正文的第一个字开始起刀。

## 三、 下一章（第 X+1 章）正文 100% 全量无损流转规范

为了彻底根治本地脚本越权、机械卡字数导致的剧情断层问题，本流水线废除一切在本地针对第 $X+1$ 章的清洗和截断代码。

### 1. 零剪裁流转状态

* **物理操作**：本地 Python 脚本仅通过标准 I/O 句柄（如 `open(f"第X+1章.txt", "r")`）一次性读入下一章的完整文本。
* **流转通道**：不经过任何字数统计、不经过任何正则表达式清洗，将其作为一个名为 `下一章前瞻背景全本文本` 的纯净字符串变量，直接塞进发给大模型 API 的请求 Payload 中。

### 2. 为什么必须 100% 全量发送？（大模型智能承载任务说明）

将第 $X+1$ 章全本文本完完整整扔给大模型的核心目的，就是为了压榨和利用大模型的全量语义理解能力，让它在后台独自完成以下三项高阶导演任务：

* **智能降噪**：大模型自己用智能去分辨第 $X+1$ 章开头是否存在"作者请假单"、"上期回顾"、"催更通知"等冗余废话，并在心算时自动将其判定为零信息量并抹除。
* **全景前瞻**：大模型通读第 $X+1$ 章全本后，能够彻底掌握未来的剧情走向（例如知道第 $X+1$ 章的中段会发生惊天大爆炸）。
* **悬念锁死**：有了对未来世界线的全面掌控，大模型在设计当前第 $X$ 章最后一集的 `画面视觉钩子` 时，能给出最具有大片质感和因果宿命感的特写定格，将漫剧的连贯性拉到最满。
