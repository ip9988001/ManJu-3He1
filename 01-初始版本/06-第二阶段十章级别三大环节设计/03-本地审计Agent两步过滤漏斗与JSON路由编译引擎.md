### 1. 第一道大闸：本地名称与特征字符串硬匹配（0 Token 消耗）

Python 脚本在内存中建立当前百章暂存资产数组，并全量扫描全局权威历史资产库（`02-大纲/01-仙逆/历史权威资产库/`）下的 12 大类文件名。

* **【硬核比对逻辑】**：
```python
# 伪代码执行逻辑
for temp_asset in sandbox_batch_assets:
    # 1. 完美撞车校验（含版本后缀）
    if temp_asset.filename in global_historical_repo[temp_asset.category]:
        # 名字与历史版本完全一致，判定为完全复用，打上复用标签，直接拦截
        temp_asset.audit_status = "EXACT_REUSE"
        temp_asset.target_path = global_historical_repo[temp_asset.category][temp_asset.filename].absolute_path
        continue

    # 2. 基础名撞车校验（去除版本号，如 检测到 飞剑_v2 与 历史库的 飞剑_v1）
    base_name = remove_version_suffix(temp_asset.filename)
    if base_name in global_historical_repo_base_names[temp_asset.category]:
        # 属于历史已有资产的潜在进化形态，打上争议标签，送往第二道大闸
        temp_asset.audit_status = "SUSPECTED_EVOLUTION"
        temp_asset.history_refer_path = global_historical_repo_base_names[temp_asset.category][base_name].absolute_path
        continue

    # 3. 模糊字符串相似度预筛（防近义词起歪，如 恒岳派灰袍 与 恒岳派灰色弟子服）
    similarity = calculate_levenshtein(temp_asset.filename, global_historical_repo_names[temp_asset.category])
    if similarity > 0.7:
        # 属于高度疑似近义词错别字，打上争议标签，送往第二道大闸
        temp_asset.audit_status = "SUSPECTED_SYNONYM"
        temp_asset.history_refer_path = get_highest_similarity_path(temp_asset.filename)
        continue

    # 4. 若以上皆未命中，判定为全新资产
    temp_asset.audit_status = "PURE_NEW"

```

### 2. 第二道大闸：低成本 LLM 1对1 决策树盲审（外科手术式判定）

针对第一道大闸筛选出来的 `SUSPECTED_EVOLUTION`（疑似升级）与 `SUSPECTED_SYNONYM`（疑似近义词）资产，Agent 启动低成本模型（`deepseek-v4-flash` 非思考模式），进行双文本 1对1 局部对决。

#### 【审计 Agent 专用 1对1 盲审系统提示词（System Prompt）】

```markdown
# 剧组资产铁面审计官指令

你现在是好莱坞级别的数字剧组资产审计判官。你的唯一任务是对比【当前新产生的资产描述】与【历史已存在的资产描述】，帮系统做出绝对精确的去重或升级裁决。

### 判定决策树核心规则：
1. **完全复用 (COMPLETE_REUSE)**：新描述与历史描述在核心视觉（容貌特征、衣服材质、道具功能）上高度一致，新描述只是多了一些不影响主体一致性的过场动作描写。
2. **形态升级 (ASSET_EVOLUTION)**：新描述对应的实体由于剧情发生重大的、不可逆的版本跃迁。
   - 角色：年龄暴长、突破境界导致容貌异变、断臂战损。
   - 道具：法宝附魔升级、外观流光特效由低阶变高阶、彻底破碎。
3. **近义抹除 (SYNONYM_ERASE)**：新产出的资产名字与历史名字不同（例如"恒岳派灰袍"与"恒岳派灰色弟子服"），但本质上100%是同一个物理积木。

### 约束要求：
必须严格以标准的 JSON 格式输出，不要包含任何多余的解释、Markdown 强调标识或 regular prose。

### 输出 JSON 字典格式规范：
{
  "action": "COMPLETE_REUSE" | "ASSET_EVOLUTION" | "SYNONYM_ERASE",
  "reason": "一句话说明判定核心理由",
  "recommended_name": "如果是近义抹除，请给出历史已有的权威标准名称；如果是升级，请给出加上新版本号的标准名（如 飞剑_v3）"
}

```

## 二、 近义词与错别字自动"改名归宗"与抹除逻辑

为了让系统具备自我纠偏能力，当第二道大闸的 LLM 返回判定结果后，本地 Agent 会执行极其刚性的**物理清理与更名动作**。

### 1. 执行 `SYNONYM_ERASE`（近义抹除）逻辑

* **触发场景**：大模型盲放了 `[恒岳派灰袍]`，历史库中存在 `[恒岳派灰色弟子服]`，LLM 返回 `"action": "SYNONYM_ERASE"`。
* **Agent 物理动作**：
1. 立即放行。Agent 直接把这一轮暂存区里的 `恒岳派灰袍.md` 物理文件**无情删除**。
2. 提取 LLM 返回的 `"recommended_name": "恒岳派灰色弟子服"`。
3. 在内存路由缓存表里登记一条强制重定向死命令：`"恒岳派灰袍" -> "历史权威资产库/02-服装造型/恒岳派灰色弟子服.md"`。

### 2. 执行 `COMPLETE_REUSE`（完全复用）逻辑

* **触发场景**：名字撞车或描述雷同，LLM 返回 `"action": "COMPLETE_REUSE"`。
* **Agent 物理动作**：
1. 判定当前暂存卡片属于无意义冗余信息，**直接在本地硬盘抹除该暂存文件**。
2. 路由指针死死扣在历史已有文件的绝对路径上。

### 3. 执行 `ASSET_EVOLUTION`（形态升级）逻辑

* **触发场景**：主角从 8 岁长到 18 岁，或天逆珠多了一道纹路，LLM 返回 `"action": "ASSET_EVOLUTION"`。
* **Agent 物理动作**：
1. 保留该资产卡片。
2. 提取 `"recommended_name": "char_wanglin_v2_18岁态"`，将该暂存卡片的文件名重命名为该标准格式。
3. 打上晋升标识，准备并入全局主干。

## 三、 资产晋升机制与 `00-本轮资产绝对路径路标集.json` 虚拟路标集编译引擎

在 300 个资产全量审计擦洗完毕后，百章周期正式宣告完结。此时系统启动最后一环：**资产向主干晋升**并**为下游脚本编译高精虚拟路标集**。

### 1. 权威主干晋升机制（Asset Promotion）

所有通过了审计、被打上 `PURE_NEW`（纯全新登场）与 `ASSET_EVOLUTION`（形态升级）标签的暂存资产卡片，由 Python 脚本执行物理迁移（`shutil.move`），从各章节的 `临时资产沙盒暂存区/` 中彻底拔出，整整齐齐地码放到全局唯一的 `历史权威资产库/` 对应的大类目录下。

至此，这些新资产正式拥有了历史编制，成为供后续千万章节随时调用的"历史老本"。
