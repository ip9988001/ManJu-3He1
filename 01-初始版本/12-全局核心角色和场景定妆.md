## 一、 核心功能与战略作用

### 1. 具体功能：【天选长相盲盒筛选】与【本地物资物理扣留】

在整部小说（或当前大卷）开工前，由人工或脚本调用线上生图 API，针对原著核心角色（如王林、墨大夫）和标志性主场景（如后山破庙），一次性生成高精度的**多角度全方位三视图**。一旦筛选出最符合原著气质的那张图，立刻执行本地物理下载。

### 2. 战略作用：终结线上模型“脸盲症”的刚性底牌

线上生图 API 最大的痛点是每次生成的脸都随机。本环节产出的“本地定妆照”，将作为下游【步骤 04：房间二（画师）】开工时的**最高母本**。Python 脚本调 API 时，会像发微信附件一样把这张本地真图直接甩过去，强制线上大模型“看着这张脸来画单集分镜”，从而在通用线上 API 架构下，实现长相 90% 以上的惊人一致性。

## 三、 投入的原材料（进门带什么）

由于这个动作**每个角色或场景一生只做一次**，所以进门带的材料必须是原著里的硬干货：

1. **原料 A：原著人设视觉侧写**：从小说正文里抠出来的关于该角色外貌、标志性衣服、身材、年龄的刚性描写。
2. **原料 B：全局核心画风参数**：从第二阶段大纲里继承下来的、这部小说指定的视觉大风格（如：正统古典写实国风）。
