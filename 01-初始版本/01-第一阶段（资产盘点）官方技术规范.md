# 漫剧制作系统：第一阶段（资产盘点）官方技术规范

## 一、 阶段核心目的 (Core Objective)

针对长篇小说连载项目（以首发 10 章，约 5 万字为基准），利用 DeepSeek API 进行“扫盲式人口普查与资产审计”。

* **核心核心**：只做“名词拉单子”，**只统计分类与名字/术语**，绝对不允许大模型在此阶段产生任何长篇大论的描述。
* **风控价值**：将长线连载中最危险的“世界观脱节”、“角色服装前后不一”、“设定吃书”等隐患，在第一步通过低成本的名词清单死死锁住。为后续的“精细参数提炼”和“全剧资产续接”夯实底层数据库结构。

---

## 二、 输入与输出边界 (I/O Boundaries)

* **系统输入**：任意题材长篇小说前 10 章纯文本（TXT），字数约 30000 - 50000 字。
* **系统输出**：完全剥离 JSON 结构的纯文本。所有名词必须被包裹在系统约定的、极其简单的**类 XML 闭合标签**内，便于本地 Python 脚本通过正则表达式进行 100% 零误差的结构化清洗与组装。

---

## 三、 12 大全题材通用资产盘点科目表

为确保系统能横扫修仙、科幻、都市、悬疑等全题材小说，DeepSeek 必须严格按照以下 12 大维度拉出名词清单：

| 序号 | 约定暗号标签 | 盘点对象与范围（全题材通用） | 漫剧流水线核心用途 |
| --- | --- | --- | --- |
| **1** | `<角色清单>` | 所有有台词、有明确动作描述的人类或拟人化角色名。 | 建立剧组演员表，锁定角色基础脸型。 |
| **2** | `<服装与造型清单>` | 明确出现的特殊服饰、面具、夜行衣、门派制服或机甲外壳。 | 实现换装功能，防止一件衣服穿到大结局。 |
| **3** | `<场景清单>` | 故事发生的所有物理地点、空间（包含室内、室外、虚拟空间）。 | 确定置景，生成生图背景的基础 Prompt。 |
| **4** | `<时间与时代清单>` | 宏观时代背景、明确的时间跨度名词、高频昼夜状态。 | 决定整体调色风格与画面的白天黑夜控制。 |
| **5** | `<核心事件清单>` | 推动剧情重大转折的冲突节点、大事件名称。 | 宏观规划分集剧本，卡准每集 70 秒高潮。 |
| **6** | `<道具与线索清单>` | 出现的武器、法宝、高科技装备、案件关键物证。 | 规范高频特写物品，保证关键道具不走样。 |
| **7** | `<技能与特殊能力>` | 角色使用的功法招式、超能力、黑科技手段或系统功能名。 | 规范特效视觉效果（如：凝聚青色斗气）。 |
| **8** | `<系统UI与特效字幕>` | 系统流/科幻流的属性面板、弹窗提示语、功法升级虚空文字。 | 制作独立的视觉贴图挂件，不与背景混淆。 |
| **9** | `<势力与组织清单>` | 故事里出现的门派、家族、财阀、帮派、政府机构。 | 批量生成群演/杂兵服装，规范势力图腾。 |
| **10** | `<非人生物与坐骑>` | 妖兽、神兽、AI实体、外星生物、异形、主角宠物。 | 针对非人类活物进行专门的怪兽类生图控制。 |
| **11** | `<世界设定清单>` | 战力等级名称、核心能源、货币单位、特殊地理界域。 | 规范全剧升级链与底层逻辑，防止模型胡编。 |
| **12** | `<环境音与卡点清单>` | 明确提到的标志性巨大声响、核心动作卡点。 | 指导 FFmpeg 自动音频剪辑，卡准音效时间戳。 |

## 五、 后端 Python 接收与 WebUI 交互逻辑

1. **正则切割**：Python 脚本利用通用的 `r"<{tag}开始>(.*?)</{tag}结束>"` 正则表达式，将 12 个模块的纯名字文本精准剥离。
2. **数据去重与清洗**：脚本自动过滤掉序号（如 `1.`、`2.`），剥离首尾空格，将干净的名字存入本地字典（Dictionary）中。
3. **导演人工介入（WebUI 呈现）**：
* WebUI 页面上，将这 12 个清单渲染为 12 个独立的**标签池/卡片流**。
* 每一个名字都是一个独立的标签。你作为总导演，可以在网页上肉眼检查。
* **动作**：一键删掉不重要的龙套名字或无意义道具；可以手动添加漏掉的名词。


4. **资产锁定**：当你点击 WebUI 上的“确认并锁定初始化资产”时，本地 Python 脚本才会将组装好的数据持久化保存（存为 `global_assets.json`），第一阶段正式完工。
