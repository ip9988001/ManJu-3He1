## ⚙️ 结合 WebUI 后，整个系统是怎么无脑运转的？

有了这个新图纸，前后端的代码对接就变得丝滑无比：

### 1. 网页开机（读账）

当你打开 WebUI 进入第 01 集页面时，前端代码会直接读取这个 `.md` 文件：

* 看到节点 01 是 `已封顶` $\rightarrow$ 网页直接加载 `image/frame_01.png`，在屏幕上展示精美画面。
* 看到节点 02 是 `渲染中` $\rightarrow$ 网页自动在屏幕上弹出一个菊花转圈图。
* 看到节点 03 是 `未渲染` $\rightarrow$ 网页显示一个灰色的虚线框。

### 2. 点击【一键生成】（改账与跑图）

你在网页上点击批量生图，后台 Python 脚本开工：

1. 先把 `.md` 里的状态无脑改成 `渲染中`。
2. 抓起本地参考图和英文 Prompt 砸给线上 API。
3. 线上图片传回来，下载到 `image/frame_02.png`。
4. Python 脚本把 `.md` 里的状态擦掉，改成 `已封顶`。
5. 网页检测到文件变了，立刻刷新，精美正片画面瞬间亮起！

### 3. 点击【Re-roll单镜重跑】（精准点杀）

如果镜头 02 的王林你觉得不满意，在网页上点击这个卡片下方的【Re-roll】：

1. 后台 Python 脚本**只定位到 `### [生图控制节点_02]` 这一块**。
2. 把它的状态改回 `渲染中`，网页上这张图立刻变成转圈状态。
3. 重新打包发送请求给线上 API。
4. 新图传回来，**直接覆盖**原有的 `frame_02.png`，随后把状态改回 `已封顶`。
5. 网页卡片一闪，新面孔的王林就换上去了，而镜头 01 和 03 稳如泰山，连动都没动！
