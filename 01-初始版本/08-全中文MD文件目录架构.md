## 二、 各级子目录功能、产出物文件及 Obsidian 联动细节

### 1. 集级核心根目录：`仙逆-第1章节-分集01/`

* **输入原料**：由步骤 02 里的 Python 物理剪刀，根据大模型的中文锚点句子，在原著中切下来的纯文本。
* **落盘文件**：`第1章01分集.txt`
* **处理动作**：作为这一集所有后续推演的“始祖源头”。

### 2. 声音车间：`01集单集声肉脱水与声卡图层提取/`

* **具体功能**：执行步骤 03。剥离原著的视觉废话，榨干出 100% 纯粹的嘴叼台词与内心独白，并给出发音人、情绪、语速调教参数。
* **落盘物料**：`01集_全角色配音与情绪调教派工单.md`
* **文件内部核心长相（Obsidian 语法规范）**：
```markdown
## 01集声音总时长预期：71秒

* 发音角色：[[旁白]]

### [声音行_02]
* 发音角色：[[王林_少年初始态]]
* 情绪音调：极度惊恐，呼吸声急促

*   **Obsidian 连线作用**：通过给角色打上 `[[王林_少年初始态]]` 的双链标签，你在 Obsidian 里点击这个角色，就能瞬间看到他在第一章第 01 集里说了哪些话、处于什么情绪，声音线一目了然！

### 3. 导演分镜车间：`01集分镜视觉图层提取/`
*   **具体功能**：执行步骤 04 的前序动作。大模型通读生肉，不考虑 AI 生图的英文提示词，**纯粹从电影导演的角度，把这一集切成一镜一镜的画面设计**，标明环境、机位、灯光、出场资产、以及对应的台词映射。
*   **落盘物料**：`01集_电影级镜头画面构图设计.md`
*   **文件内部核心长相（Obsidian 语法规范）**：
    ```markdown
    ## 01集导演分镜大盘
    
    ### [分镜镜头_01]
    * 映射声音：[声音行_01]
    * 画面景别：大远景（Extreme Long Shot）
    * 视觉构图：古老破败的修仙庙宇孤零零立在荒山之中，漫天暴雨形成白色水雾，整体色调呈压抑的青灰色。
    * 出场资产：[[环境_荒山破庙]], [[天气_特大暴雨]]
    
    ### [分镜镜头_02]
    * 映射声音：[声音行_02]
    * 画面景别：特写（Close-up）
    * 视觉构图：少年王林猛然回头，双瞳剧烈收缩，脸颊上有雨水和冷汗混杂流下，背景是古庙闪电划过的惨白亮光。
    * 出场资产：[[王林_少年初始态]], [[环境_破庙内部]]

```

* **Obsidian 连线作用**：为什么要单独拆出这个分镜文件夹？因为这是**人眼对账的核心**。你可以通过 `[[环境_荒山破庙]]` 的双链，在 Obsidian 图谱里清晰看到这个场景在哪些分镜里出现过，防止后期画面产生“场景穿帮”。

### 4. Flux 物料车间：`01集动态提示词Prompt高精编译/`

* **具体功能**：执行步骤 04 的终极动作。把上面那个通俗易懂的“导演分镜画面”，加上第二阶段沉淀在资产表里的“Flux 刚性长相英文 Prompts”，融合编译成**可以直接复制、或者由 Python 脚本直接提取去调用 Stable Diffusion / Flux API 的最终生图代码单**。

## 三、 全量 Markdown 化对本地 Python 自动化脚本的降维减负

老兄，你坚持用 `.md` 替代 JSON，在代码层面上其实完成了一次精妙的**“格式解耦”**：

1.  **解析极度弱智化**：Python 脚本去读 `.md` 文件时，再也不用担心“JSON 格式解析失败”报错了。脚本只需要用简单的行扫描：
    *   看到 `* 发音角色：`，就用 `.split("：")[1]` 拿走角色名字；
    *   看到 `* Flux 终极渲染 Prompt：`，就直接把下一行的整段英文抓走存进生图队列。
2.  **人工改账无感同步**：如果某一天你觉得大模型生成的台词不够爽，或者 Flux 的提示词需要微调，你直接在 Obsidian 软件里把这个 `.md` 文件的字改了。Python 脚本下次启动读取时，读到的就是你手工精修后的完美版本，**实现了无缝的“人机协同排产”**！
