# 系统架构重构方案：新增剧本叙事层

---

## 一、问题本质

当前系统存在的不是"缺少一个 Prompt"的问题，而是**叙事建模层级错位（Narrative Layer Misalignment）**。

当前结构：

```
小说 → 大纲 → 分镜
```

其中"大纲"同时承担了结构拆分、节奏控制和内容压缩的职责，"分镜"同时承担了剧本编写和视觉拆解的职责，中间缺少独立的"叙事执行层"。这导致 AI 在一个环节中同时做编剧、分镜师和视觉设计三件事，产出不稳定、不可控、质量波动大。

---

## 二、正确的影视工业分层标准

真实的影视工业结构为五层：

| 层级 | 名称 | 说明 |
|------|------|------|
| L0 | 原始文本（Novel/IP） | 小说原著 |
| L1 | 结构切分（Episode Breakdown） | 分集切分 |
| L2 | 单集叙事单元（Episode Bible / MD） | 叙事蓝图，核心中间层 |
| L3 | 剧本层（Screenplay） | 对白、动作、节奏 |
| L4 | 分镜层（Storyboard） | 视觉化拆解 |

当前系统缺少 L2 和 L3，直接从 L1 跳到 L4。

---

## 三、修复后的完整架构

```
① 小说原文
    ↓
② P1：题材识别 + 风格系统
    ↓
③ P2：分集切分（Episode Outline）
    ↓
④ P2.1：Episode MD 生成（单集叙事结构化）——新增核心层
    ↓
⑤ P2.2：Script 剧本生成（基于 MD）——新增核心层
    ↓
⑥ P3：分镜生成（Shot Breakdown）
    ↓
⑦ P4：图像提示词优化
    ↓
⑧ P5：视频提示词优化
```

每一层对应的人类岗位：

| 层级 | 对应岗位 |
|------|---------|
| MD 层（P2.1） | 制片人 / 剧集总策划 |
| Script 层（P2.2） | 编剧 |
| Shot 层（P3） | 分镜师 / 导演 |

---

## 四、新增核心模块 1：Episode MD Builder

### 作用

将"分集大纲"转化为可执行的单集叙事蓝图（Narrative Execution Layer）。

### 输入

- 分集大纲
- 角色库
- 全局风格
- 小说原文片段

### 输出：Episode MD 标准结构

```markdown
# Episode X - MD

## 1. 本集目标（Narrative Goal）
- 本集核心冲突是什么

## 2. 情绪曲线（Emotion Arc）
- 起 → 承 → 转 → 合

## 3. 场景列表（Scene List）
- Scene 1：引入
- Scene 2：冲突
- Scene 3：升级
- Scene 4：反转
- Scene 5：结尾钩子

## 4. 角色状态锁定（Character State Lock）
- 主角当前心理状态
- 反派动机状态
- 配角功能状态

## 5. 信息密度分配（Key Info Map）
- 哪些信息必须在本集出现

## 6. 对白风格约束（Dialogue Rules）
- 本集语言风格控制
```

---

## 五、新增核心模块 2：Script Layer（剧本层）

### 输入关系（关键修正）

Script 的输入必须是 Episode MD，而不能是大纲：

```
Script = function(Episode MD)
```

具体输入：

- 场景列表（来自 MD）
- 情绪曲线（来自 MD）
- 角色状态锁定（来自 MD）

### 输出结构

```xml
<场景>
  <对话>
  <动作>
  <情绪节奏>
  <场景调度>
</场景>
```

## 六、Prompt 链修正

旧链：

```
P1 → P2 → P3 → P4 → P5
```

新链：

```
P1：题材识别
P2：分集大纲
P2.1：Episode MD 生成（核心新增）
P2.2：剧本生成（核心新增）
P3：分镜生成
P4：图像提示词优化
P5：视频提示词优化
```

P4 和 P5 的输入因此升级为结构化剧本（而非半结构文本），包含已结构化的剧本内容、情绪曲线、场景节点，输出质量将显著提升，减少随机性。

---

## 七、WebUI Tab 3 界面修正

原来的布局：

```
左栏：小说输入、分集大纲
中栏：分镜脚本（直接从大纲跳转）
右栏：分镜预览、状态机日志
```

修正后的布局：

```
左栏：
  - 小说输入
  - 分集大纲

中栏（新增核心区域）：
  - Episode MD Viewer（核心新增，展示单集叙事蓝图）
  - Script Editor（剧本编辑器，展示完整台词和动作）
  - Scene Editor（场景编辑器）

右栏：
  - 分镜预览
  - 状态机日志
```

数据流从"大纲 → 分镜"变为"大纲 → MD → 剧本 → 分镜"。

---

## 八、数据结构修正

### SQLite 新增字段

```sql
ALTER TABLE scenes ADD COLUMN episode_md TEXT;
ALTER TABLE scenes ADD COLUMN script_text TEXT;
ALTER TABLE scenes ADD COLUMN emotion_arc TEXT;
```

### MD 文件结构升级

每个单集从一个 MD 文件拆分为三个：

```
episode_01.md            # 分集大纲
episode_01_script.md     # 剧本内容
episode_01_scene.md      # 分镜脚本
```

---

## 九、状态机修正

新状态流转：

```
idle
→ p1（题材识别）
→ p2（分集大纲）
→ episode_md_build（叙事结构生成）——新增
→ script_build（编剧层生成）——新增
→ review（人工审核）
→ shot_build（分镜生成）
→ render（合成输出）
```

---

## 十、修复收益

| 维度 | 修复前 | 修复后 |
|------|--------|--------|
| 剧本质量 | 不稳定，台词和角色容易漂移 | 情绪连续、台词稳定、角色不漂移 |
| 分镜质量 | 输入模糊，分镜模型在"猜剧本" | 输入结构明确，视觉拆解精准 |
| Prompt 4/5 质量 | 输入为半结构文本，随机性大 | 输入为结构化剧本，质量显著提升 |
| 人工审核成本 | 需要在分镜层大量修正 | 在 MD 层和 Script 层提前拦截，分镜层审核量减少 |
| 系统本质 | AI 自动写故事 + 自动拆镜头（不可控） | 工业编剧系统 + 分镜生成系统 + 多模态生产线（可控） |
