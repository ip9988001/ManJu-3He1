## 一、题材自动识别与视觉风格加载

### 1.1 功能说明

【取自系统1·P1】读取小说原文，自动判断题材类型，加载该题材对应的完整视觉风格词库。这一步为全剧定调——后续所有生成环节都围绕这个"风格锚点"来，避免画面风格混乱。

### 1.2 支持的题材类型

| 题材 | 视觉基准关键词 |
|------|---------------|
| 玄幻修仙 | 仙风道骨、飞剑流光、灵力涌动、云雾仙山、法宝灵光、天劫雷电 |
| 现代都市霸总 | 高级西装、大理石冷调、落地窗城市夜景、商务谈判桌、黑色轿车 |
| 古风言情 | 烟雨江南、油纸伞、红墙绿瓦、罗裙飘飘、玉簪花、青石长街 |
| 悬疑惊悚 | 暗巷昏黄路灯、血迹、碎裂镜子、幽暗走廊、诡异微笑、废弃医院 |
| 科幻废土 | 锈蚀金属、霓虹全息投影、废墟荒漠、机械义体、赛博义眼 |
| 都市异能 | 现代城市背景、超自然光效、能量护盾、意念操控、夜色天台 |
| 战争军旅 | 硝烟弥漫、迷彩军装、铁血战场、军营篝火、战地医疗 |
| 校园青春 | 阳光走廊、课桌椅、校服、操场、樱花树、篮球架 |
| 末日求生 | 荒废城市、防护面具、变异生物、临时营地、生存物资 |

### 1.3 输入

- 小说原文前 10 章纯文本（约 3-5 万字）

### 1.4 输出（8 大产出物）

| 产出物 | 说明 | 在下游的作用 |
|--------|------|-------------|
| 题材标签 | 如"玄幻修仙" | 决定全剧视觉基调 |
| 混合题材说明 | 如主线为修仙+副线为言情 | 处理题材交叉情况 |
| 视觉词库 | 该题材核心视觉元素列表 | 后续所有生成的风格锚点 |
| 色彩基调 | 如"冷青、雪白、暗金为主色调" | 全局调色依据 |
| 光影风格 | 如"云层体积光、冷月逆光" | 生图光影控制 |
| 镜头语言 | 如"低机位仰拍、中近景压迫构图" | 分镜构图依据 |
| 服装道具词库 | 该题材常见服饰道具 | 防止服装风格串味 |
| 环境场景词库 | 该题材常见场景 | 防止场景穿帮 |
| 禁用混入元素 | 如修仙禁止汽车/枪械/霓虹灯 | 防止现代元素污染古代题材 |

### 1.5 输出格式

```xml
<题材标签>玄幻修仙</题材标签>
<混合题材说明>主线为宗门覆灭与剑修复仇，情感线尚未明显出现。</混合题材说明>
<视觉词库>仙风道骨、飞剑流光、灵力涌动、云雾仙山、法宝灵光</视觉词库>
<色彩基调>以冷青、雪白、暗金为主色调，辅以灵力流光和残火橙光。</色彩基调>
<光影风格>云层体积光、冷月逆光、法宝灵光、闪电瞬间强光。</光影风格>
<镜头语言>低机位仰拍、中近景压迫构图、远景展示宗门废墟。</镜头语言>
<服装道具词库>玄色破损道袍、月白云纹长袍、银灰腰封、断剑。</服装道具词库>
<环境场景词库>破败山门、雪夜石阶、云雾仙山、宗门废墟。</环境场景词库>
<禁用混入元素>现代高楼、汽车、枪械、赛博机械、校园校服。</禁用混入元素>
<题材置信说明>文本以修仙宗门为核心场景，判定为主题材。</题材置信说明>
```

### 1.6 红线约束

| 红线 | 说明 |
|------|------|
| 禁止空泛词 | 视觉词库必须是具体可视化词汇，禁止"好看""高级""震撼" |
| 禁止输出代码 | 只输出 XML 标签，不输出 JSON/Markdown/Python |
| 混合题材处理 | 主题材写前面，副题材写后面 |
| 禁用元素必须列全 | 严禁遗漏，防止后续生图出现不合规元素 |

### 1.7 人工审核点

✅ 建议审核——确认题材判断是否准确、视觉词库是否合适、禁用元素是否完整。

### 1.8 WebUI 呈现

在 WebUI 的"资产盘点"Tab 中展示：
- 题材标签显示在项目配置区
- 视觉词库以标签云形式展示
- 禁用元素以红色警告标签展示
- 人工可以增删修改

## 二、十二大资产科目盘点

### 2.1 功能说明

【取自系统2】对小说前 10 章进行"扫盲式人口普查"，只拉名词清单，不写长篇描述。把长线连载中最危险的"世界观脱节"、"角色服装前后不一"、"设定吃书"等隐患，在第一步通过低成本的名词清单死死锁住。

### 2.2 核心原则

**只做"名词拉单子"，绝对不允许大模型在此阶段产生任何长篇大论的描述。** 这一步的目的是建立"户口本"，不是写"人物传记"。

### 2.3 十二大科目详细说明

#### 科目 01：核心角色清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 所有有台词、有明确动作描述的人类或拟人化角色名 |
| 包含 | 主角、配角、反派、有台词的龙套 |
| 不包含 | 仅被提及但未出场的角色 |
| 漫剧用途 | 建立剧组演员表，锁定角色基础脸型 |
| 举例 | 王林、墨大夫、张虎、李慕婉 |

#### 科目 02：服装与造型清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 明确出现的特殊服饰、面具、夜行衣、门派制服或机甲外壳 |
| 包含 | 有专门描写的衣服、制服、面具、盔甲 |
| 不包含 | 普通日常衣物（如"穿着衣服"这种笼统描述） |
| 漫剧用途 | 实现换装功能，防止一件衣服穿到大结局 |
| 举例 | 恒岳派灰色记名弟子服、墨大夫黑色道袍、战神殿血袍 |

#### 科目 03：场景空间清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 故事发生的所有物理地点、空间（包含室内、室外、虚拟空间） |
| 包含 | 有明确描写的地点、建筑、自然环境 |
| 不包含 | 仅被提及但角色从未到达的地点 |
| 漫剧用途 | 确定置景，生成生图背景的基础 Prompt |
| 举例 | 恒岳派后山破庙、神手谷药田、修魔海坊市 |

#### 科目 04：时间与时代清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 宏观时代背景、明确的时间跨度名词、高频昼夜状态 |
| 包含 | 时代背景、时间段落、标志性昼夜 |
| 漫剧用途 | 决定整体调色风格与画面的白天黑夜控制 |
| 举例 | 传统东方古风、残阳伴暮色、深夜暴雨、拂晓时分 |

#### 科目 05：核心事件清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 推动剧情重大转折的冲突节点、大事件名称 |
| 包含 | 打斗、背叛、捡宝、突破、死亡、复活等关键事件 |
| 不包含 | 日常过渡性事件 |
| 漫剧用途 | 宏观规划分集剧本，卡准每集 70 秒高潮 |
| 举例 | 意外捡到神秘绿瓶、墨大夫悬崖对决、王林筑基突破 |

#### 科目 06：道具与线索清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 出现的武器、法宝、高科技装备、案件关键物证 |
| 包含 | 有专门描写或多次出现的物品 |
| 不包含 | 一次性提及的普通物品 |
| 漫剧用途 | 规范高频特写物品，保证关键道具不走样 |
| 举例 | 神秘小绿瓶、天逆珠、生锈柴刀、下品灵石 |

#### 科目 07：技能与特殊能力清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 角色使用的功法招式、超能力、黑科技手段或系统功能名 |
| 包含 | 有名称、有具体效果描述的能力 |
| 不包含 | 笼统的"施法""出招" |
| 漫剧用途 | 规范特效视觉效果（如凝聚青色斗气） |
| 举例 | 罗烟步、引力术、长春功、极境杀戮神光 |

#### 科目 08：系统UI与特效字幕清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 系统流/科幻流的属性面板、弹窗提示语、功法升级虚空文字 |
| 包含 | 系统面板、属性窗口、升级提示、警告文字 |
| 漫剧用途 | 制作独立的视觉贴图挂件，不与背景混淆 |
| 举例 | 属性面板、长春功突破层数提示框、系统商城界面 |

#### 科目 09：势力与组织清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 故事里出现的门派、家族、财阀、帮派、政府机构 |
| 包含 | 有名称、有成员出场的组织 |
| 不包含 | 仅被提及但从未有成员出场的组织 |
| 漫剧用途 | 批量生成群演/杂兵服装，规范势力图腾 |
| 举例 | 七玄门、野狼帮、战神殿、恒岳派 |

#### 科目 10：非人生物与坐骑清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 妖兽、神兽、AI实体、外星生物、异形、主角宠物 |
| 包含 | 有名称、有具体形态描写的非人角色 |
| 漫剧用途 | 针对非人类活物进行专门的怪兽类生图控制 |
| 举例 | 双头暴烈犬、铁奴巨汉傀儡、神秘石门守卫 |

#### 科目 11：世界设定清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 战力等级名称、核心能源、货币单位、特殊地理界域 |
| 包含 | 有名称的等级体系、货币、能量、地理概念 |
| 漫剧用途 | 规范全剧升级链与底层逻辑，防止模型胡编 |
| 举例 | 练气期/筑基期/元婴期、低阶灵石、修仙界/凡人界 |

#### 科目 12：环境音与卡点清单

| 维度 | 说明 |
|------|------|
| 盘点范围 | 明确提到的标志性巨大声响、核心动作卡点 |
| 包含 | 有具体描写的声音事件（如"喀嚓一声巨响"） |
| 漫剧用途 | 指导 FFmpeg 自动音频剪辑，卡准音效时间戳 |
| 举例 | 飞剑出鞘金铁交鸣音、天劫雷电轰鸣、法宝碰撞爆破声 |

### 2.5 后端 Python 处理流程

1. **正则切割**：用 `r"<{tag}开始>(.*?)</{tag}结束>"` 正则表达式，将 12 个模块的纯名字文本精准剥离
2. **数据去重与清洗**：自动过滤序号（1.、2.），剥离首尾空格，存入本地字典
3. **WebUI 呈现**：12 个清单渲染为 12 个独立的标签池/卡片流
4. **人工审核**：你作为总导演在网页上肉眼检查，一键删掉不重要的龙套，手动添加漏掉的名词
5. **资产锁定**：点击"确认并锁定初始化资产" → Python 持久化保存为 `global_assets.json`

### 2.6 红线约束

| 红线 | 说明 |
|------|------|
| 只拉名词 | 绝对不允许大模型在此阶段写长篇描述 |
| 标签格式 | 只用中文尖括号标签，禁止 JSON/Markdown |
| 必须全标签 | 12 个标签一个都不能省略，无法判断的输出空标签对 |
| 禁止出现代码 | 不准输出 Python、HTML、JavaScript |

### 2.7 人工审核点
