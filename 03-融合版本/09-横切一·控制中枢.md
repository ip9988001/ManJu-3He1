## 一、任务状态机

### 1.1 功能说明

【取自系统1】控制整个生产流水线的"大脑"。管理从"待处理"到"生产完成"的全链路状态流转。

### 1.2 状态定义

```python
class SceneStatus(Enum):
    PENDING = "待处理"           # 等待开始
    LLM_GENERATING = "LLM生成中"  # 大模型正在生成剧本/分镜/提示词
    WAITING_REVIEW = "等待人工审核" # 等待人类确认
    APPROVED = "已确认"           # 人工审核通过
    GENERATING_IMAGE = "图像生成中" # 正在调用图像API
    GENERATING_VIDEO = "视频生成中" # 正在调用视频API
    GENERATING_AUDIO = "音频生成中" # 正在调用TTS/BGM API
    COMPOSING = "合成中"          # 正在用FFmpeg合成
    DONE = "生产完成"             # 成片输出
    FAILED = "失败"              # 某环节出错
    PAUSED = "暂停"              # 人工暂停
```

### 1.3 状态流转图

```
[*] → PENDING
PENDING → LLM_GENERATING
LLM_GENERATING → WAITING_REVIEW
WAITING_REVIEW → APPROVED（人工通过）
WAITING_REVIEW → PENDING（人工驳回，重来）
APPROVED → GENERATING_IMAGE
GENERATING_IMAGE → GENERATING_VIDEO
GENERATING_VIDEO → GENERATING_AUDIO
GENERATING_AUDIO → COMPOSING
COMPOSING → DONE

GENERATING_IMAGE → FAILED
GENERATING_VIDEO → FAILED
GENERATING_AUDIO → FAILED
FAILED → PENDING（重试）
任何状态 → PAUSED（人工暂停）
PAUSED → 任意状态（恢复）
```

### 1.4 核心数据结构

```python
@dataclass
class SceneTask:
    scene_id: str
    episode_id: str
    status: str = "PENDING"
    retry_count: int = 0
    created_at: float = field(default_factory=time.time)
    payload: Dict[str, Any] = field(default_factory=dict)
    task_id: str = field(default_factory=lambda: str(uuid.uuid4()))
```

## 二、任务调度器

### 2.1 功能说明

【取自系统1】控制任务的并发执行，支持优先级和限流。

### 2.2 实现

```python
class TaskScheduler:
    def __init__(self, gateway, concurrency=5):
        self.gateway = gateway
        self.queue = deque()
        self.running = {}
        self.semaphore = asyncio.Semaphore(concurrency)

    def add_task(self, task):
        self.queue.append(task)

    async def run(self):
        while True:
            if not self.queue:
                await asyncio.sleep(0.5)
                continue
            task = self.queue.popleft()
            asyncio.create_task(self._execute(task))

    async def _execute(self, task):
        async with self.semaphore:
            try:
                task.status = SceneStatus.LLM_GENERATING
                await self.run_llm(task)
                task.status = SceneStatus.WAITING_REVIEW
                await self.wait_for_review(task)
                task.status = SceneStatus.APPROVED
                await self.run_pipeline(task)
                task.status = SceneStatus.DONE
            except Exception as e:
                task.status = SceneStatus.FAILED
```

## 三、断点续传

### 3.1 功能说明

【取自系统1】系统崩了可以从上次中断的地方继续，不需要从头来。

### 3.2 实现

```python
class CheckpointManager:
    def __init__(self, path="checkpoint.json"):
        self.path = path

    def save(self, task):
        data = {
            "task_id": task.task_id,
            "status": task.status,
            "payload": task.payload
        }
        with open(self.path, "w") as f:
            json.dump(data, f)

    def load(self):
        if not os.path.exists(self.path):
            return None
        with open(self.path, "r") as f:
            return json.load(f)
```

### 3.3 恢复流程

1. 系统重启
2. CheckpointManager.load() 读取上次状态
3. 如果状态是"GENERATING_VIDEO"，从视频生成环节继续
4. 不需要重新执行 LLM 生成和图像生成

## 五、成本监控系统

### 5.1 功能说明

【取自系统1】记录每次 API 调用的成本，按类别统计。

### 5.2 实现

```python
class CostTracker:
    def __init__(self):
        self.cost_map = defaultdict(float)

    def add_cost(self, provider: str, cost: float):
        self.cost_map[provider] += cost

    def get_total(self):
        return dict(self.cost_map)

    def log(self):
        print("\n=== COST REPORT ===")
        for k, v in self.cost_map.items():
            print(f"{k}: ${v:.4f}")
```

## 六、双轨滚动控制

### 6.1 功能说明

【取自系统2】管理宏观盘点和微观精制的交替前进。

### 6.2 控制逻辑

```python
class DualTrackController:
    def __init__(self):
        self.current_chapter = 0
        self.macro_locked = False

    async def run(self):
        while self.current_chapter < self.total_chapters:
            # 轨A：宏观盘点（每10章触发）
            if self.current_chapter % 10 == 0 and not self.macro_locked:
                await self.run_macro_inventory()
                await self.wait_for_asset_lock()
                self.macro_locked = True

            # 轨B：微观精制（逐章）
            await self.run_micro_production(self.current_chapter)
            self.current_chapter += 1

            # 每100章触发审计
            if self.current_chapter % 100 == 0:
                await self.run_sandbox_audit()
                self.macro_locked = False
```

## 七、异常处理

### 7.1 错误分类

```python
class APIException(Exception): pass
class RetryableError(APIException): pass    # 可重试（429、500、超时）
class FatalError(APIException): pass        # 不可重试（401、403）
class RateLimitError(RetryableError): pass  # 限流
class AuthError(FatalError): pass           # 认证失败
class InvalidRequestError(FatalError): pass # 请求格式错误
```

### 7.2 重试策略

| 错误类型 | 策略 |
|----------|------|
| RateLimitError | 等待 Retry-After 时间后重试 |
| RetryableError | 指数退避重试（2^attempt 秒），最多 3 次 |
| FatalError | 不重试，直接报错 |
| 重试 3 次后仍失败 | 触发降级链路切换备选 API |
