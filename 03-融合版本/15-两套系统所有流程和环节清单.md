# 两套漫剧制作系统 —— 全流程全环节盘点汇总

---

## 📋 第一套系统：漫剧/短剧自动化生产系统

### 总流程链路

```
小说原文 → 题材识别 → 分集大纲 → 叙事蓝图 → 剧本 → 分镜脚本 → 提示词精炼 → 多媒体素材生成 → 自动剪辑合成 → 成片输出
```

### 全部流程环节清单

| 序号 | 阶段 | 环节 | 功能 |
|------|------|------|------|
| 1 | 题材识别 | P1：题材自动识别与视觉风格词库加载 | 判断小说题材，加载对应视觉风格词库 |
| 2 | 题材识别 | 色彩基调 / 光影风格 / 镜头语言确定 | 为全剧定调色、定光影、定镜头风格 |
| 3 | 题材识别 | 服装道具词库 / 环境场景词库生成 | 建立该题材的服装和场景"字典" |
| 4 | 题材识别 | 禁用混入元素锁定 | 禁止出现不符合题材的元素（如修仙里禁止汽车） |
| 5 | 分集大纲 | P2：分集大纲生成 | 小说切分成适合短剧节奏的分集大纲 |
| 6 | 分集大纲 | 核心冲突 / 情绪走向 / 结尾钩子设计 | 每集有冲突、高潮、悬念 |
| 7 | 分集大纲 | 角色资产初稿生成 | 首次出场角色的完整外貌描述 |
| 8 | 叙事蓝图 | P2.1：Episode MD 生成（单集叙事结构化） | 补充层——将大纲转化为可执行的叙事蓝图 |
| 9 | 叙事蓝图 | 情绪曲线 / 场景列表 / 角色状态锁定 | 确保故事结构扎实、节奏正确 |
| 10 | 叙事蓝图 | 信息密度分配 / 对白风格约束 | 控制每集信息量和语言风格 |
| 11 | 剧本生成 | P2.2：Script 剧本生成 | 补充层——基于叙事蓝图生成完整剧本 |
| 12 | 剧本生成 | 逐场景对白 / 动作描写 / 情绪节奏标注 | 确保台词精彩、角色不漂移 |
| 13 | 分镜脚本 | P3：逐镜分镜脚本生成 | 剧本拆解为 26 字段的结构化分镜 |
| 14 | 分镜脚本 | 角色外貌 / 角色锚定词 / 情绪 / 动作 / 服装道具 | 锁定每一镜的角色状态 |
| 15 | 分镜脚本 | 场景环境 / 光影构图 / 镜头语言 / 画面风格 | 锁定每一镜的画面设计 |
| 16 | 分镜脚本 | 文生图提示词 / 反向提示词 | 直接喂给 AI 画图的完整提示词 |
| 17 | 分镜脚本 | 视频动态提示词 / 运动限制 / 镜头时长 | 直接喂给 AI 做视频的提示词 |
| 18 | 分镜脚本 | 台词配音 / 情绪标签 / 角色音色 | 配音指令 |
| 19 | 分镜脚本 | BGM提示词 / 音效提示词 | 音乐和音效指令 |
| 20 | 分镜脚本 | 转场建议 / 一致性备注 / 人工审核提示 | 剪辑和防崩指令 |
| 21 | 提示词精炼 | P4：文生图提示词精炼 | 优化图像提示词（补全细节、删空泛词） |
| 22 | 提示词精炼 | P5：视频动态提示词精炼 | 优化视频提示词（防崩坏、运动强度校准） |
| 23 | 素材生成 | 图像生成（文生图） | 调用 Kling/Flux/Seedream 等生成图片 |
| 24 | 素材生成 | 视频生成（图生视频） | 调用 Kling/Runway/Seedance 等生成视频 |
| 25 | 素材生成 | 配音生成（TTS） | 调用 ElevenLabs/MiniMax/Edge-TTS 生成配音 |
| 26 | 素材生成 | 背景音乐生成（BGM） | 调用 Suno/Udio/MusicGen 生成音乐 |
| 27 | 素材生成 | 音效生成（SFX） | AI 生成或音效库检索匹配 |
| 28 | 素材生成 | 自动降级链路（Fallback） | 某个 API 挂了自动切换备选 |
| 29 | 自动剪辑 | 素材下载与缓存 | 云端素材下载到本地 |
| 30 | 自动剪辑 | 静态图 → 动态视频（Ken Burns 效果） | 让静态图片"动起来" |
| 31 | 自动剪辑 | 音画同步 | 配音与视频画面对齐 |
| 32 | 自动剪辑 | BGM 混音 | 配音 + BGM + 音效三层混合 |
| 33 | 自动剪辑 | 字幕生成（ASS 格式） | 根据台词生成工业级字幕 |
| 34 | 自动剪辑 | 单分镜合成 | 一个分镜的所有素材合成一个片段 |
| 35 | 自动剪辑 | 整集拼接 | 一集所有分镜按顺序拼接 |
| 36 | 成片输出 | 最终视频文件（MP4） | 可播放的短剧成片 |
| 37 | 成片输出 | 项目级 Markdown 文件 | 记录所有分镜完整信息 |
| 38 | 成片输出 | 处理日志 | 每个环节的状态、耗时、成本 |

### 横向支撑系统

| 序号 | 模块 | 功能 |
|------|------|------|
| S1 | API 网关层 | 统一接口、自动重试、并发控制、成本记录、可插拔适配器 |
| S2 | 任务调度 + 状态机 | 全链路状态流转、断点续传、人工审核 Hook |
| S3 | 角色一致性控制 | Seed / LoRA / FaceSwap 三种策略自动选择 |
| S4 | WebUI 控制台 | 项目管理、角色资产库、剧本工作台、素材生产台、合成输出台、成本监控 |
| S5 | Markdown 文件系统（SSOT） | LLM 输出 → 解析 → Markdown → 人工审核 → 后续环节读取 |
| S6 | XML 解析器 | LLM 输出的 XML 标签 → 标准字典 → 校验必填字段 |
| S7 | Prompt Engine | 5 阶段 Prompt 模板加载与执行 |
| S8 | 任务队列 | async + Semaphore 并发调度 |
| S9 | 数据系统 | SQLite 索引 + 文件系统存储 |
| S10 | 部署系统 | Docker + FastAPI + Gradio |
