## 一、叙事蓝图生成（新增层）

### 1.1 功能说明

【取自系统1·P2.1】将分集大纲转化为可执行的单集叙事蓝图。填补"大纲→分镜"之间的空白层，确保故事结构扎实、节奏正确。

### 1.2 为什么需要这一层？

没有叙事蓝图时，大模型同时当编剧+分镜师+视觉设计师，质量不稳定。补上这一层后：
- 叙事蓝图 = 确保故事结构扎实、节奏正确
- 剧本 = 确保台词精彩、角色不漂移
- 分镜只需要做"视觉拆解"这一件事

### 1.3 输入

| 输入 | 来源 |
|------|------|
| 分集大纲 | 十章视听控制总纲中的 `[Episodic_Cut_&_Hooks]` |
| 角色库 | 12大资产中的核心角色清单 |
| 全局风格 | 题材识别的视觉词库 + 色彩基调 |
| 小说原文片段 | 当前集对应的原著切片 |

### 1.4 输出格式

```markdown
# Episode X - 叙事蓝图

## 1. 本集目标（Narrative Goal）
- 本集核心冲突是什么
- 本集必须推进的剧情节点

## 2. 情绪曲线（Emotion Arc）
- 开场情绪：紧张/平静/压抑
- 中段转折：冲突升级/信息揭露
- 高潮爆发：情绪最高点
- 结尾收束：悬念留钩

## 3. 场景列表（Scene List）
- Scene 1：引入（建立场景+角色状态）
- Scene 2：冲突（核心矛盾爆发）
- Scene 3：升级（事态恶化或反转）
- Scene 4：反转（出乎意料的转折）
- Scene 5：结尾钩子（让观众想看下一集）

## 4. 角色状态锁定（Character State Lock）
- 主角当前心理状态：如"极度戒备但内心恐惧"
- 反派动机状态：如"正在执行计划的第三步"
- 配角功能状态：如"提供关键情报后退场"

## 5. 信息密度分配（Key Info Map）
- 哪些信息必须在本集出现
- 哪些信息必须在本集隐藏

## 6. 对白风格约束（Dialogue Rules）
- 本集语言风格：古风文雅/现代口语/冷冽简洁
- 情绪台词占比：如60%紧张对白+30%内心独白+10%环境旁白
```

### 1.5 人工审核点

✅ 必须审核——确认叙事结构、情绪曲线、场景列表、对白风格。

## 三、微观单章输入组装

### 3.1 功能说明

【取自系统2】当排产到第 X 章时，Python 脚本在发给大模型前组装好 5 大素材。

### 3.2 五大素材合流

| 图层 | 素材 | 来源 | 作用 | 容量 |
|------|------|------|------|------|
| 图层1 | 系统刚性教条 | Python底层常数 | 锁死"1秒=4汉字"、单集240-360字 | ~600字 |
| 图层2 | 中观导演手册 | 十章视听控制总纲 | 告诉大模型这章是大战章还是过渡章 | ~800字 |
| 图层3 | 后顾接力棒 | 上一章JSON派工单 | 从上一章最后一镜无缝接续 | ~400字 |
| 图层4 | 当期原料 | 当前章TXT（已切肉） | 正文素材 | 1,000-8,000字 |
| 图层5 | 前瞻预判灯塔 | 下一章TXT（100%全量） | 让大模型知道未来走向 | 3,000-10,000字 |

### 3.3 物理切肉算法

**【取自系统2】** 根据上一章的"吞噬终点句子"，在当前章开头物理切掉已被吃剩的内容。

| 步骤 | Python 动作 | 异常兜底 |
|------|------------|----------|
| A：清洗去噪 | `.strip()` 剔除前后空格换行 | 标记线为空则熔断报错 |
| B：全文检索 | `.find(吞噬标记线)` | 返回 -1 则报警转人工 |
| C：计算切点 | `cut_point = index + len(标记线)` | 锁死最后标点符号 |
| D：物理截断 | `切剩正文 = 全文[cut_point:]` | 送入内存作为核心原料 |

### 3.4 创世块冷启动

**【取自系统2】** 处理第一章时，因为没有"上一章"，在内存中硬编码假数据防止崩溃。

```
上一集分集ID：创世集_零号索引
上一集音频台词钩子：大幕拉开，万物初始。
上一集画面视觉钩子：电影级全黑转场，冷调色，画面中无任何出场角色。
上一集残余环境白噪音：-60dB（绝对静音状态）
跨章吞噬机制：是否激活=否，吞噬终点句子=前情空白
```

### 3.5 下一章 100% 全量无损流转

【取自系统2】不做任何字数统计、正则清洗，将下一章完整文本直接塞进 API 请求。让大模型自己用智能过滤噪点、寻找跨章反转爆点。

## 五、Python 物理切肉与 JSON 派工单编译

### 5.1 功能说明

【取自系统2】Python 根据大模型指定的"文本终点锚点"句子，在原著中精确定位切割，编译 JSON 派工单。

### 5.2 执行逻辑

1. 提取分集总数
2. 遍历每一集标签块
3. 按行提取字段（锚点/钩子/资产）
4. 在原著中定位锚点句子，物理切割
5. 跨章场景：本章碎肉 + 下一章鲜肉自动缝合
6. 单集物料落盘为 `ep_XX_raw.txt`
7. 编译 `第X章_流水线生产总执行派工单.json`

### 5.3 JSON 派工单格式

```json
{
  "目标章号": 1415,
  "本章动态分集总数": 3,
  "分集详细生产大盘": [
    {
      "分集序号": "ep_01",
      "单集生肉文本路径": "第1415章微观排产区/ep_01_raw.txt",
      "配音台词钩子文案": "墨大夫，这是你逼我的！",
      "画面视觉分镜构图": "局部特写，主角右手狠狠拍在储物袋上，绿芒喷薄而出。",
      "本集关键资产探针": ["王林_少年初始态", "墨大夫"],
      "跨章滑动焊接层": {
        "是否激活跨章吞噬": false,
        "下一章物理截止句子": ""
      }
    }
  ]
}
```

### 5.4 落盘结构


## 六、声音车间：声肉脱水与配音派工

### 6.1 功能说明

【取自系统2】剥离原著中所有视觉废话，只保留 100% 纯粹的配音内容。为自动化并发配音提供"绝对无歧义"的执行工单。

### 6.2 输入

| 输入 | 来源 |
|------|------|
| 分集原著生肉 | ep_01_raw.txt |
| 控制常数 | 上一步的台词钩子 + 临界资产 |
| 中观导演手册 | 十章视听控制总纲 |
| 标签合同 | 全中文标签输出合同（System Prompt） |

### 6.3 输出格式

```
<开始单集声图提取>
有效配音总字数：280

<开始配音行_1>
发音角色：旁白
情绪音调：【低沉引入】冷调悬疑，语速放慢15%，营造大幕拉开的窒息感。
有效声音内容：荒山，古庙。暴雨如注，砸在青苔乱石之上，溅起漫天水雾。
<结束配音行_1>

<开始配音行_2>
发音角色：王林_少年初始态
情绪音调：【极度惊恐】呼吸声急促，带有一丝少年特有的颤音。
有效声音内容：谁？！是谁在庙后说话？！给老子滚出来！
<结束配音行_2>

<开始配音行_3>
发音角色：墨大夫
情绪音调：【沙哑讥笑】喉音共鸣严重，带着胜券在握的残忍。
有效声音内容：王林，你当真以为，你能逃出老夫的五指山吗？
<结束配音行_3>
<结束单集声图提取>
```

### 6.4 四大刚性红线

| 红线 | 说明 | 违反后果 |
|------|------|----------|
| 纯声音红线 | 有效声音内容里严禁夹带动作描述（不允许"王林：（惊恐地退后两步）谁？！"） | 格式污染导致 TTS 引擎出错 |
| 角色实名归宗 | 必须用资产表标准实名（如"王林_少年初始态"），不准用"王林""少年" | Python 找不到 Voice ID |
| 零对白特判 | 如果整集全是武打无台词，强制全量判定为"旁白" | 不输出标签导致流水线卡死 |
| 尾尾咬合 | 本集最后一行配音文本必须与上一步定好的"音频台词钩子"100%逐字对齐 | 章节切换时悬念断裂 |

### 6.5 多角色异口同声特判

如果原著出现"众人齐声惊呼"：
- 严禁合并吐出"发音角色：众人"（因为本地找不到众人的 Voice ID）
- 必须强制归类为系统通用音色"旁白_龙套群音"
- 或者拆成多行输出

### 6.6 下游自动变现

**变现 A：自动计算全集总时长**
```
全集视频总长度 = 有效配音总字数 ÷ 4字/秒
例：280字 ÷ 4 = 70秒
```

**变现 B：多线程 TTS 批量配音**
1. 读到"发音角色：王林_少年初始态" → 自动匹配 Voice_ID_WangLin_001
2. 读到"情绪音调：极度惊恐" → 自动调整语速、声调、稳定性
3. 连同纯文本台词一起打包发给 TTS 引擎
4. 几秒钟后 audio/ 文件夹里躺着 line_01.mp3、line_02.mp3...

### 6.7 人工审核点

✅ 建议审核——确认配音行的情绪标注是否准确、角色分配是否正确。
