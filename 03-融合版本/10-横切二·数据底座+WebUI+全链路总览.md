## 一、全中文 Markdown 文件系统

### 1.1 功能说明

【取自系统2】整个系统的"唯一真理来源"。所有数据以 `.md` 文件形式存储。

### 1.2 为什么用 Markdown 不用 JSON？

| 维度 | Markdown | JSON |
|------|----------|------|
| Python 解析 | 极简：`.split("：")[1]` 提取字段 | 需要 json.loads，格式错误就崩 |
| 人工编辑 | 用记事本就能改 | 多一个逗号就报错 |
| 可视化 | 文件管理器里一眼看清 | 需要专门工具 |
| Obsidian 联动 | 双链标签 `[[王林]]` | 不支持 |
| 防崩能力 | 即使格式不完美也能解析 | 严格格式要求 |

### 1.3 文件命名规范

- 全中文命名
- 禁止空格（用下划线替代）
- 禁止特殊字符
- 序号前缀保证排序正确

### 1.4 Obsidian 双链标签

通过 `[[角色名]]` 双链标签：
- 点击角色名 → 看到他在哪些分镜里出现过
- 在 Obsidian 图谱中可视化资产之间的关系
- 防止后期画面产生"场景穿帮"

## 三、SQLite 索引系统

### 3.1 功能说明

【取自系统1】任务状态、成本记录的结构化查询，支持断点续传。

### 3.2 Schema

```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    name TEXT,
    genre TEXT,
    aspect_ratio TEXT DEFAULT '9:16',
    status TEXT,
    created_at TEXT
);

CREATE TABLE episodes (
    id INTEGER PRIMARY KEY,
    project_id INTEGER,
    chapter_number INTEGER,
    episode_number INTEGER,
    md_path TEXT,
    status TEXT,
    created_at TEXT
);

CREATE TABLE tasks (
    id INTEGER PRIMARY KEY,
    type TEXT,
    status TEXT,
    input TEXT,
    output TEXT,
    cost REAL,
    created_at TEXT
);

CREATE TABLE assets (
    id INTEGER PRIMARY KEY,
    category TEXT,
    name TEXT,
    version INTEGER,
    md_path TEXT,
    png_path TEXT,
    created_at TEXT
);
```

## 五、WebUI 控制台完整设计

### 5.1 技术选型

Gradio Blocks + FastAPI 后端。Gradio 做操作台，FastAPI 做后端调度。

### 5.2 七大 Tab

#### Tab 1：📁 项目管理

| 组件 | 功能 |
|------|------|
| 项目列表 | 展示所有项目（名称/题材/集数/进度/更新时间） |
| 新建项目 | 输入项目名称、原著名称、题材、目标集数 |
| 画幅配置 | 9:16 / 16:9 / 1:1 |
| 一致性策略 | Seed / LoRA / FaceSwap / Auto |
| 模型选择 | LLM / 图像 / 视频 / TTS 各选一个 |

#### Tab 2：🧍 选角大厅

| 组件 | 功能 |
|------|------|
| 角色列表 | 展示所有已盘点角色 |
| 定妆照展示 | 已定妆角色显示高清照，未定妆显示灰色剪影 |
| 盲盒选角 | 点击后 API 轰出 4 张三视图供选择 |
| 一键换角 | 重新海选并覆盖替换 |
| 角色编辑 | 外貌描述、Voice ID、LoRA ID、参考图 |

#### Tab 3：📋 资产盘点

| 组件 | 功能 |
|------|------|
| 12 大标签池 | 每个科目一个标签流/卡片流 |
| 人工增删 | 删除不重要的龙套、手动添加漏掉的名词 |
| 确认锁定 | 点击后写入 global_assets.json |

#### Tab 4：📝 剧本工作台

| 区域 | 组件 | 功能 |
|------|------|------|
| 左栏 | 小说输入 + Prompt 控制 | 输入原文，点击 P1/P2/P2.1/P2.2/P3 按钮 |
| 中栏 | 分镜编辑器 | 展示 15 字段分镜，可逐字段编辑 |
| 右栏 | 预览 + 日志 + 状态 | 实时预览、运行日志、状态机当前状态 |

**Prompt 事件链：**
```
P1 题材识别 → P2 分集大纲 → P2.1 叙事蓝图 → P2.2 剧本 → P3 分镜生成
→ P4 文生图精炼 → P5 视频动态精炼 → 英文编译 → 生成 MD
```

#### Tab 5：🎬 素材生产台

| 区域 | 组件 | 功能 |
|------|------|------|
| 左栏 | 分镜列表 | Checkbox 选择要生产的分镜 |
| 中栏 | 生产控制 | 一键生产图片/视频/配音/BGM/全部 |
| 右栏 | 预览 | WebUI 渲染状态联动（未渲染/渲染中/已封顶） |

**并发执行：**
```python
async def run_assets(scene_ids):
    semaphore = asyncio.Semaphore(5)
    async def worker(scene):
        async with semaphore:
            await generate_image(scene)
            await generate_video(scene)
            await generate_tts(scene)
```

#### Tab 6：🎞 合成输出台

| 组件 | 功能 |
|------|------|
| 集数选择 | 选择要合成的集数 |
| 草稿预览 | 快速预览（低质量） |
| 开始合成 | 正式合成（高质量） |
| 进度条 | 实时显示合成进度 |
| 视频预览 | 合成完成后直接播放 |

#### Tab 7：💰 成本监控

| 组件 | 功能 |
|------|------|
| LLM 成本 | 文字显示 |
| 图像成本 | 文字显示 |
| 视频成本 | 文字显示 |
| 语音成本 | 文字显示 |
| 总成本 | 文字显示 |
| 成本趋势图 | 按天/周/月的变化曲线 |
