## 一、提示词精炼（新增层）

### 1.1 功能说明

【取自系统1·P4/P5】对分镜中的文生图提示词和视频动态提示词进行二次精炼优化。系统2直接从中文分镜跳到英文提示词，中间没有精炼环节，补上后显著提升质量。

### 1.2 文生图提示词精炼

**精炼方向：**

| 方向 | 具体操作 |
|------|----------|
| 补全角色细节 | 加入面部解剖学参数（单眼皮狭长眼、黝黑哑光皮肤、灰色麻绳扎马尾） |
| 补全环境细节 | 加入三层景物布局（前景枯藤/中景青石台/背景纯黑阴影） |
| 补全光影信息 | 加入主光源角度（左上45度斜射冷白丁达尔光束） |
| 补全镜头信息 | 加入景别、机位、角度、景深 |
| 删除空泛词 | 删掉"很美""很帅""很震撼""很有感觉" |
| 确保一致性 | 不改变角色核心外貌特征 |

**质量标准：**
- 长度控制在 120-260 个中文字符
- 必须是"客观画面描述"，不是"对模型的请求"
- 必须包含：主体 + 环境 + 光影 + 构图 + 风格 + 质量 + 禁止项

### 1.3 视频动态提示词精炼与防崩规则

**【取自系统1·P5】** 核心解决 AI 视频生成时的画面崩坏问题。

**六大禁止动作：**

| 禁止动作 | 为什么禁止 | 替代方案 |
|----------|-----------|----------|
| 快速奔跑 | 容易导致人物结构漂移 | 镜头加速推进，衣物更明显飘动 |
| 飞速冲向 | 容易导致画面变形和瞬移 | 推镜头，中景转近景 |
| 猛地跳起 | 容易导致肢体崩坏 | 身体重心轻微变化，衣袂上扬 |
| 高速旋转 | 容易导致脸部和背景崩坏 | 低速环绕镜头 |
| 连续打斗 | 多动作难稳定 | 武器光效闪烁、指节收紧、眼神变冷 |
| 变身换装 | 破坏角色一致性 | 光效增强但服装和五官不变 |

**运动强度校准表：**

| 情绪标签 | 运动强度 | 推荐动态 | 推荐镜头运动 |
|----------|----------|----------|-------------|
| 虚弱/悲伤/平静 | 低 | 微表情变化、呼吸起伏、雪花飘落 | 柔和拉远、浅景深 |
| 紧张/悬疑/恐惧 | 中 | 缓慢推镜、阴影移动、灯光闪烁 | 近景推镜、嘴角细微变化 |
| 愤怒/激动/战斗 | 高 | 推镜头稍快、衣物强烈飘动、能量光效增强 | 推镜头稍快、环境动态增强 |
| 温柔/怀念/释然 | 低 | 柔和拉远、发丝微动、光斑流动 | 浅景深、缓慢后拉 |

**优化示例：**

优化前：
```
萧玄愤怒地冲向黑影，拿剑疯狂攻击，雷电爆炸。
```

优化后：
```
镜头以稍快速度向前推进，中景逐渐压缩为近景。
萧玄保持单膝跪地姿势，右手指节缓缓收紧，符文断剑上的青色残光逐层亮起。
玄色破损道袍和半束黑发在寒风中明显翻动。
远处雷光短暂照亮坍塌山门，血色积雪表面掠过冷蓝反光。
萧玄的眼神由压抑痛楚转为冷冽杀意，画面保持肃杀和稳定。
运动强度等级：高
```

### 1.4 人工审核点

✅ 必须审核——确认精炼后的提示词质量、视频动态是否符合防崩规则。

## 二、视觉车间·房间二：英文提示词编译 + 定妆照路径

### 2.1 功能说明

【取自系统2·房间二】将精炼后的中文分镜翻译编译为英文提示词，并指定本地定妆照的物理路径。

### 2.2 输入

| 输入 | 来源 | 作用 |
|------|------|------|
| 精炼后的分镜 | 来自提示词精炼环节 | 加工原料 |
| 全局资产路径账本 | 跨章继承的全局角色资产数据库 | 记录所有角色的本地定妆照绝对路径 |
| 动态全局画风表 | 十章视听控制总纲 | 当前章节规定的画面画风与核心色调英文词 |

### 2.3 输出格式

```markdown
# 01集 线上生图通用 API 代码料单
* 状态标签：#画师编译车间已封顶
* 依据分镜：[[01集_电影级镜头画面构图设计.md]]

### [生图控制节点_01]
* WebUI渲染状态: 已封顶
* 附加本地参考图: None
* 本地正片输出路径: 仙逆-第1章节-分集01/image/frame_01.png
* 线上生图核心Prompt: (Cinematic long shot, masterpiece, 8k resolution, photorealistic), 
  established ancient Chinese style, dark teal and gray color tone, 
  an ancient dilapidated temple standing isolated on a dark mountain 
  in a heavy thunderstorm, thick water mist, fierce lightning flashes --ar 16:9

### [生图控制节点_02]
* WebUI渲染状态: 渲染中
* 附加本地参考图: 03-全局核心角色定妆库/角色_王林_少年初始态/王林_天选定妆图.png
* 本地正片输出路径: 仙逆-第1章节-分集01/image/frame_02.png
* 线上生图核心Prompt: (Cinematic close-up shot, intense emotion, highly detailed skin texture), 
  established ancient Chinese style, a 15-year-old Chinese boy, thin face, 
  sharp eyebrows, black long hair tied up, wearing grey ancient robes, 
  eyes dilated in extreme fear, sweat mixed with rain trickling down --ar 16:9

### [生图控制节点_03]
* WebUI渲染状态: 未渲染

* 本地正片输出路径: 仙逆-第1章节-分集01/image/frame_03.png
* 线上生图核心Prompt: (Cinematic medium shot, dynamic pose, high contrast), 
  established ancient Chinese style, an elderly Chinese doctor, gaunt sallow face, 
  sharp hawk-like eyes, wearing a loose black Taoist robe, a cruel smile --ar 16:9
```

### 2.4 三大刚性红线

| 红线 | 说明 |
|------|------|
| 路径死锁 | 附加本地参考图路径必须与全局资产账本100%像素级一致，错一个字母Python找不到图就熔断 |
| 数量一比一 | 中文分镜输入了几个镜头就必须编译出几个生图控制节点，严禁合并或删减 |
| 线性绝对不去重 | 同一角色多个镜头，每个都要完整写路径，严禁写"同上" |

### 2.5 Python 自动化执行逻辑

1. 扫描料单，定位 `### [生图控制节点_XX]`
2. 读取 `* 附加本地参考图:` 后面的物理路径
3. 去硬盘把真正的 `.png` 图片文件读进内存
4. 把图片 + 英文提示词一起打包发给线上 API（像发微信附件）
5. 线上模型看着真图画，出来的角色长相相似度 90%+
6. 返回成品链接，Python 下载命名为 `frame_XX.png`

### 2.6 WebUI 联动

料单中的状态钩子驱动 WebUI：
- `未渲染` → 网页显示灰色虚线框占位符
- `渲染中` → 网页显示菊花转圈图
- `已封顶` → 网页显示精美成品图

用户点击"Re-roll"→ Python 只定位到该节点，改回"渲染中"，重新打包发送，新图覆盖旧图，其他镜头不受影响。

### 2.7 人工审核点

✅ 必须审核——确认英文提示词质量、定妆照路径正确、画风一致。

## 三、统一 API 网关

### 3.1 功能说明

【取自系统1】所有外部 API 调用的统一入口，解决 6 个核心问题。

### 3.2 六大核心能力

| 能力 | 说明 |
|------|------|
| 统一接口 | 不管调 Kling 还是 Runway，调用方式一样 |
| 异步/同步统一 | LLM 同步、视频生成异步轮询 |
| 自动重试 + 指数退避 | 失败了自动重试，每次等待时间翻倍（2^attempt 秒） |
| 并发控制 | asyncio + Semaphore 控制同时调用几个 API |
| 成本记录 | 每次 API 调用记录 provider、cost、timestamp |
| 可插拔适配器 | 新增 API 只需写一个适配器文件，继承 BaseAdapter |

### 3.3 适配器架构

```
api_gateway/
├── base/
│   ├── base_adapter.py      # 统一基类（send_request/check_status/fetch_result/handle_error）
│   ├── exceptions.py        # APIException/RetryableError/FatalError/RateLimitError/AuthError
│   ├── retry.py             # 指数退避重试
│   └── limiter.py           # Token Bucket 限流器
├── adapters/
│   ├── llm/                 # DeepSeek/Kimi/OpenAI
│   ├── image/               # Kling/Flux/Seedream
│   ├── video/               # Runway/Luma/Seedance
│   ├── tts/                 # ElevenLabs/CosyVoice/MiniMax
│   └── music/               # Suno/Udio/MusicGen
├── core/
│   ├── gateway.py           # 核心调度器（register_adapter/call）
│   ├── task_manager.py
│   └── cost_tracker.py      # 成本追踪（add_cost/get_total/log）
└── config.py
```

### 3.4 统一调用示例

```python
gateway = APIGateway()
gateway.register_adapter("kling", KlingAdapter(config), rate_limit=10)
gateway.register_adapter("runway", RunwayAdapter(config), rate_limit=5)

# 统一调用
result = await gateway.call("kling", prompt="...", image_url="...")
```

## 四、自动降级链路

### 4.1 功能说明

【取自系统1】某个 API 挂了自动切换备选，永远不停。

### 4.2 降级配置

```python
FALLBACK_CHAIN = {
    "kling_image": ["flux", "seedream", "dalle3"],
    "kling_video": ["runway", "seedance", "luma"],
    "elevenlabs": ["cosyvoice", "minimax", "edge_tts"],
    "suno": ["udio", "musicgen"],
    "deepseek": ["kimi", "openai"],
}
```

### 4.3 降级执行

```python
async def safe_generate(gateway, provider, func, *args, **kwargs):
    try:
        return await getattr(gateway.adapters[provider], func)(*args, **kwargs)
    except Exception:
        for fallback in FALLBACK_CHAIN.get(provider, []):
            try:
                return await getattr(gateway.adapters[fallback], func)(*args, **kwargs)
            except:
                continue
        raise
```

## 五、角色一致性四级策略

### 5.1 功能说明

【系统2定妆照 + 系统1三策略融合】确保同一个人物在不同镜头里长相一致。

### 5.2 四级优先级

| 优先级 | 策略 | 来源 | 原理 | 适用场景 |
|--------|------|------|------|----------|
| 1 | 本地定妆照 + 真图直传 | 系统2 | 把本地.png连同提示词一起发给API | 默认首选 |
| 2 | LoRA 角色模型 | 系统1 | 训练角色专属模型 | 长剧、高一致性需求 |
| 3 | Seed 固定随机种子 | 系统1 | 固定随机种子 | 简单场景 |
| 4 | FaceSwap 后处理 | 系统1 | 先生成再换脸 | 高动态场景 |

### 5.3 自动策略选择器

```python
def select_strategy(character_id, scene_complexity):
    if has_local_portrait(character_id):
        return "portrait_reference"
    if scene_complexity == "high_motion":
        return "faceswap"
    if scene_complexity == "long_episode":
        return "lora"
    return "seed"
```

## 六、全局定妆系统

### 6.1 功能说明

【取自系统2】在整部剧开机前，为核心角色和场景生成定妆照，存入本地硬盘。

### 6.2 定妆照生成公式

```
(Character design sheet, turnaround, multiple angles of the same character, 
front view, side view, back view, detailed facial expressions), 
[角色外貌侧写英文], [全局画风参数英文], 
studio white background, clean lineart, cinematic lighting, photorealistic --ar 16:9
```

### 6.3 定妆大账格式

```markdown
# 角色_王林_少年初始态_全方位定妆大账
* 状态标签：#全局核心角色资产已封顶

### [WebUI 选角控制中心]
* WebUI定妆状态: 已定锁
* 物理真图绝对路径: 03-全局核心角色定妆库/角色_王林_少年初始态/王林_天选定妆图.png
* 黄金种子号Seed: 999999999

## 1. 线上生图唯一指定：天选外貌黄金 Prompt 母本
(Character design sheet, turnaround, multiple angles), 
a 15-year-old Chinese boy, thin face, sharp eyebrows...

## 2. 角色出场统计与下游联动规则
* 联动规则：下游扫描到本角色时，必须抓取本地真图以多模态附件发包。
```

### 6.4 WebUI 选角大厅交互

| 状态 | WebUI 显示 | 用户操作 |
|------|-----------|----------|
| 未定妆 | 灰色剪影 + "虚位以待"按钮 | 点击"开始盲盒选角" |
| 盲盒渲染中 | 转圈等待 | 等待 API 返回 4 张照片 |
| 已定锁 | 高清定妆照大图 | 可点击"重新海选"换角 |

### 6.5 一键换角流程

1. 点击"重新海选"
2. 账本状态改为"盲盒渲染中"
3. API 轰出 4 张新照片
4. 鼠标点击选中，点击"就用这张"
5. 新图下载覆盖替换旧 `.png`
6. 账本状态改回"已定锁"
7. 下游拍正片时抓到的就是全新面孔
